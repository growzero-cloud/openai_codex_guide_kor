00

시작하기 전에

처음 목표는 코드를 전부 맡기는 것이 아니라, 안전하게 읽히고 작은 작업부터 검증하는 것입니다.

이 가이드의 대상

Codex를 한 번도 써본 적 없는 사람입니다. 터미널이 낯설어도 App이나 IDE Extension으로 시작하면 됩니다.

핵심 관점

Codex는 코딩 답변을 주는 챗봇이 아니라, 프로젝트 폴더를 읽고 파일 수정과 검증까지 도와주는 코딩 에이전트입니다.

  • 처음에는 파일 수정 없이 읽기와 설명부터 시킵니다.
  • 작업 전후로 Git 상태를 확인합니다.
  • 민감정보, 운영 데이터, 결제, 권한 정책은 신중하게 다룹니다.
01

Codex란?

Codex는 OpenAI의 소프트웨어 개발용 코딩 에이전트입니다. 코드 작성, 이해, 리뷰, 디버깅, 자동화를 도와줍니다.

일반 ChatGPTCodex
대화창에서 답변을 받음프로젝트 폴더를 읽고 실제 변경을 제안하거나 수행
코드를 붙여넣어 질문저장소 구조, 관련 파일, 테스트 명령을 탐색
사용자가 결과를 복사해 적용diff, 테스트, 리뷰, PR 흐름까지 연결

04

설치와 로그인

가장 쉬운 시작은 Codex App입니다. 터미널이 편하다면 CLI를 설치해도 됩니다.

Codex App 설치 (macOS / Windows)

  1. chatgpt.com/codex 또는 공식 Quickstart 페이지에서 본인 운영체제(macOS / Windows)용 앱을 다운로드합니다.
  2. 다운로드된 파일(.dmg 또는 .exe)을 열고 설치합니다.
  3. 앱을 처음 실행하면 로그인 화면이 나타납니다. ChatGPT 계정(OpenAI 계정)으로 로그인합니다.
  4. 로그인 후 "Open a project folder" 버튼을 클릭해 작업할 프로젝트 폴더를 선택합니다. (없으면 빈 폴더를 새로 만들어 선택해도 됩니다.)
  5. 왼쪽 파일 트리와 오른쪽 채팅창이 보이면 준비 완료입니다.
처음엔 권한을 좁게 시작하세요

앱 첫 실행 시 권한 설정 화면이 나오면 Read-only 또는 기본값으로 시작하세요. 나중에 필요할 때 넓힐 수 있습니다.

IDE Extension 설치 (VS Code 기준)

  1. VS Code를 엽니다.
  2. 왼쪽 사이드바에서 Extensions 아이콘(또는 Ctrl+Shift+X) 클릭.
  3. 검색창에 "Codex" 입력 → 게시자가 OpenAI인 확장 선택.
  4. Install 클릭 → 설치 완료.
  5. 왼쪽 사이드바에 새로 생긴 Codex 아이콘 클릭.
  6. Sign in 버튼 클릭 → 브라우저가 열리면 OpenAI 계정으로 승인.
  7. VS Code로 돌아오면 Codex 패널에 채팅창이 표시됩니다.
다른 에디터

Cursor, Windsurf, JetBrains도 각 마켓플레이스에서 "Codex" 검색 후 동일 방식으로 설치합니다.

CLI 설치 (터미널 사용자용)

Node.js 18 이상이 설치된 환경에서 실행합니다.

npm install -g @openai/codex

macOS Homebrew:

brew install codex

로그인

처음 실행 시 API key 또는 ChatGPT 계정 로그인 화면이 나타납니다.

codex
CLI는 터미널이 익숙한 분께 권장합니다

처음이라면 App 탭으로 돌아가세요.

첫 실행: 만들기 전에 읽히기

처음부터 수정하지 말고 프로젝트 구조, 실행 명령, 테스트 명령을 먼저 설명하게 합니다.

Codex App에서 첫 실행하기

  1. Codex App을 열고 프로젝트 폴더를 선택합니다.
  2. 오른쪽 채팅창에 아래 프롬프트를 붙여넣고 전송합니다.
  3. Codex가 파일을 읽고 프로젝트 구조를 설명합니다. 수정은 하지 않습니다.
  4. 설명이 맞으면 AGENTS.md 초안 요청으로 이어갑니다.
이 프로젝트를 처음 온 사람 기준으로 설명해줘.

수정하지 말고 다음만 알려줘:
1. 프로젝트 목적
2. 주요 폴더
3. 실행 명령
4. 테스트 명령
5. 중요한 설정 파일
6. 처음 작업할 때 주의할 점

설명이 맞으면 이어서 요청합니다.

이 내용을 바탕으로 AGENTS.md 초안을 만들어줘.
아직 파일은 만들지 말고 먼저 초안만 보여줘.
06

프롬프트 기본기

Codex 프롬프트는 목표, 맥락, 제약, 완료 기준 네 가지로 쓰면 안정적입니다.

목표:
- 무엇을 바꾸거나 만들고 싶은지

맥락:
- 관련 파일, 폴더, 에러 메시지, 화면, 재현 방법

제약:
- 바꾸면 안 되는 것
- 새 의존성 추가 여부
- 스타일이나 아키텍처 규칙

완료 기준:
- 테스트 통과
- 특정 동작 확인
- 문서 업데이트

08

Plan Mode

여러 파일, 인증, 결제, 마이그레이션, 큰 리팩터링은 바로 구현하지 말고 계획부터 세웁니다.

App에서 Plan Mode 사용 방법

  1. Codex App 채팅창에서 큰 작업을 시작하기 전, 아래처럼 계획만 먼저 요청합니다.
  2. Codex가 단계별 계획, 수정 예상 파일, 리스크를 표시합니다.
  3. 계획을 검토하고 OK면 "계획대로 진행해줘"로 이어갑니다. 수정이 필요하면 수정 지시를 먼저 합니다.
먼저 계획만 세워줘.

목표:
회원가입 플로우에 이메일 인증 단계를 추가하고 싶어.

산출물:
- 단계별 계획
- 수정 예상 파일
- 리스크
- 내가 먼저 결정해야 할 질문

아직 파일 수정은 하지 마.
언제 Plan Mode를 쓰나요?

여러 파일 동시 수정, 인증/결제 로직 변경, 대규모 리팩터링, 마이그레이션처럼 되돌리기 어려운 작업 전에 항상 계획을 먼저 받으세요.

09

AGENTS.md

AGENTS.md는 Codex가 매번 읽는 작업 지침서입니다. 실행 명령, 테스트, 금지사항, 완료 기준을 저장합니다.

AGENTS.md 만드는 방법

  1. Codex App에서 프로젝트 폴더를 연 뒤, 채팅창에 "AGENTS.md 초안을 만들어줘. 파일은 아직 만들지 말고 보여만 줘."라고 입력합니다.
  2. Codex가 초안을 보여주면 내용을 검토합니다.
  3. 수정할 부분을 말하고 최종본 확정 후 "이제 프로젝트 루트에 AGENTS.md 파일로 저장해줘."라고 요청합니다.
  4. 파일이 생성되면 diff를 확인하고 커밋합니다.

AGENTS.md 예시:

# AGENTS.md

## 실행 명령
- 개발 서버: `npm run dev`
- 테스트: `npm test`
- 빌드: `npm run build`

## 작업 규칙
- 새 의존성 추가 전 사용자에게 확인합니다.
- 대규모 리팩터링은 먼저 계획을 제시합니다.
- 민감정보는 출력하거나 커밋하지 않습니다.

## 완료 기준
- 관련 테스트를 실행합니다.
- 변경 파일과 검증 결과를 요약합니다.

CLI에서는 /init으로 초안을 만들 수 있습니다.

10

권한과 샌드박스

Codex는 파일과 명령어를 다루므로 sandbox mode와 approval policy를 이해해야 합니다.

App에서 권한 설정 방법

  1. Codex App 상단 메뉴 또는 설정(⚙️) 아이콘 클릭.
  2. Permissions 또는 Sandbox 항목 선택.
  3. 처음에는 Read-only로 설정합니다. 분석과 읽기 작업은 이 모드로 충분합니다.
  4. 파일 수정이 필요해지면 workspace-write로 변경합니다.
  5. Full Access는 신뢰된 환경에서 고급 작업 시에만 사용합니다.
모드설명초보자 추천
Read-only읽기 중심. 수정과 명령 실행은 제한처음 분석할 때
Auto / workspace-write작업 폴더 안에서 읽기, 수정, 명령 실행 가능일반 작업
Full Access넓은 접근과 네트워크를 허용신뢰된 환경에서만
 
/permissions
주의

Full Access는 편하지만 위험합니다. 초보자는 기본 권한 또는 Read-only에서 시작하세요.

12

모델과 reasoning 선택

2026-05-25 기준 공식 문서에서 Codex의 일반적인 권장 모델은 gpt-5.5입니다.

App에서 모델 바꾸는 방법

  1. Codex App 채팅창 상단 또는 설정에서 모델 선택기를 클릭합니다.
  2. 작업 난이도에 맞는 모델을 선택합니다. 모르면 기본값(gpt-5.5)으로 시작하세요.
  3. reasoning 강도도 같은 화면에서 low / medium / high로 선택 가능합니다.
gpt-5.5복잡한 코딩, 계획, 디버깅, 도구 사용의 기본 선택
gpt-5.4일반 로컬 작업에서 사용량을 아끼고 싶을 때
gpt-5.4-mini간단하고 반복적인 로컬 메시지
gpt-5.3-CodexCodex 특화 작업, Cloud task와 GitHub code review 영역

reasoning은 작은 작업은 low, 일반 작업은 medium, 복잡한 디버깅은 high부터 시작합니다.

13

Git과 함께 쓰기

Codex는 Git과 함께 쓸 때 안전합니다. 작업 전후 상태를 확인하고 작은 단위로 검토하세요.

App에서 Git 상태 확인 방법

  1. Codex App 왼쪽 패널 또는 상단 탭에서 Git / Changes 탭을 클릭합니다.
  2. 변경된 파일 목록과 diff가 표시됩니다. 작업 전후로 반드시 확인하세요.
  3. 작업이 끝나면 채팅창에 아래 리뷰 프롬프트를 입력합니다.
현재 변경사항을 리뷰해줘.
우선순위는 버그, 보안, 테스트 누락, 유지보수성 순서로 봐줘.

MCP

MCP(Model Context Protocol)는 Codex가 외부 도구와 컨텍스트에 연결되는 표준입니다.

App에서 MCP 연결 방법

  1. Codex App 상단 설정(⚙️) 클릭 → MCP 탭 선택.
  2. Add MCP Server 클릭.
  3. MCP 서버 이름과 실행 명령을 입력합니다. (예: context7 → npx -y @upstash/context7-mcp)
  4. 저장 후 채팅창에서 MCP 도구를 사용할 수 있습니다.

문서

OpenAI Docs, Context7 같은 최신 문서 검색 — 최신 API 확인에 유용

디자인

Figma 디자인을 읽어 컴포넌트 코드 생성

브라우저

Playwright나 Chrome DevTools로 실제 화면 검사 및 테스트 자동화

CLI에서 MCP 추가 (선택)

codex mcp add context7 -- npx -y @upstash/context7-mcp

Skills

Skill은 반복되는 작업 방식을 SKILL.md로 포장한 것입니다. Codex가 같은 절차를 안정적으로 따르게 합니다.

Skill 만드는 방법

  1. Codex App 채팅창에 아래처럼 Skill 생성을 요청합니다.
  2. Codex가 SKILL.md 초안을 보여줍니다. 내용을 검토합니다.
  3. 확정 후 "이 Skill을 .claude/skills/[이름]/SKILL.md로 저장해줘"라고 요청합니다.
  4. 다음부터 같은 작업이 생기면 Codex가 자동으로 이 Skill을 참고합니다.
이 작업 패턴을 Skill로 만들어줘.

my-skill/

  • 팀 공통 리뷰 기준이나 릴리스 절차가 있을 때

자동화

Codex App에서는 반복 작업을 일정에 맞춰 백그라운드로 실행할 수 있습니다.

Automation 만드는 방법

  1. Codex App 왼쪽 사이드바에서 Automations 또는 Schedules 탭 클릭.
  2. New Automation 클릭.
  3. 실행할 프롬프트, 대상 프로젝트, 반복 일정(매일/매주 등)을 설정합니다.
  4. Save → 설정한 시간에 자동 실행됩니다. 결과는 앱 알림 또는 이메일로 수신.
  • 매일 최근 커밋 요약
  • 주기적 버그 가능성 스캔
  • 릴리스 노트 초안
  • CI 실패 확인
  • PR 상태 점검
순서

보안 수칙

Codex는 강력한 도구입니다. 민감정보와 운영 데이터는 초보 단계부터 엄격히 다뤄야 합니다.

  • API key, token, password, private key를 대화에 직접 붙여넣지 않습니다.
  • .env 값을 문서나 로그에 남기지 않습니다.
  • 이해하지 못한 설치 스크립트 실행을 승인하지 않습니다.
  • 운영 DB, 결제, 고객 데이터 수정 요청은 사전 승인 없이는 하지 않습니다.
  • Full Access를 습관적으로 사용하지 않습니다.
이 작업 중 민감정보가 발견되면 원문을 출력하지 말고 ***REDACTED***로만 표시해줘.
API key, token, password, cookie, private key는 코드나 문서에 남기지 마.
